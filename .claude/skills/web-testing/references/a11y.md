# Accessibility testing — deep dive

Bake a11y checks into E2E flows, not as a separate suite. They catch regressions where keyboard focus, ARIA names, or contrast quietly degrade.

## 1. Tool choice

- **`@axe-core/playwright`** for Playwright E2E — full WCAG ruleset, scoped to a `page` or `locator`.
- **`@axe-core/react` (or `jest-axe`)** for Vitest component tests — runs axe on a rendered RTL tree.
- **Manual screen-reader passes** for new flows — VoiceOver (macOS), NVDA (Windows). Tools catch ~30% of issues.

## 2. Playwright integration

```ts
import AxeBuilder from '@axe-core/playwright';
import { expect, test } from '@playwright/test';

test('extraction page passes WCAG 2.1 AA', async ({ page }) => {
  await page.goto('/extraction/123');
  await page.waitForLoadState('networkidle');

  const results = await new AxeBuilder({ page })
    .withTags(['wcag2a', 'wcag2aa', 'wcag21a', 'wcag21aa', 'best-practice'])
    .exclude('[data-testid="third-party-widget"]')
    .analyze();

  expect(results.violations).toEqual([]);
});
```

### Scoping

```ts
// Test one component:
new AxeBuilder({ page }).include('[data-testid="hitl-config-form"]')

// Exclude noisy third-party:
.exclude('.recharts-tooltip')

// Disable specific rules with justification:
.disableRules(['color-contrast'])  // ticket #321 — design system in progress
```

Justify every `.disableRules` call with a ticket. Otherwise the bar erodes.

## 3. Vitest component a11y

```tsx
import { axe, toHaveNoViolations } from 'jest-axe';
import { render } from '@testing-library/react';

expect.extend(toHaveNoViolations);

it('PublishDialog is accessible', async () => {
  const { container } = render(<PublishDialog open onClose={() => {}} />);
  const results = await axe(container);
  expect(results).toHaveNoViolations();
});
```

Use this for primitive components (dialogs, menus, custom inputs) before they ship. For composed pages, rely on Playwright + axe — jsdom doesn't render CSS so it misses contrast and focus-visible violations.

## 4. WCAG rules we care about most

| Rule (axe ID)               | What it catches                                                    | Why it bites us                                                                |
|-----------------------------|--------------------------------------------------------------------|--------------------------------------------------------------------------------|
| `color-contrast`             | < 4.5:1 contrast for normal text, < 3:1 for large                  | shadcn defaults are close; custom tokens (`text-muted-foreground/40`) often fail. |
| `aria-required-attr`         | Missing required ARIA attrs (e.g. `aria-expanded` on toggles)      | Hand-rolled Radix wrappers forget this.                                        |
| `label`                       | `<input>` without an accessible name                                | Forms built ad-hoc skip `<Label>` or `aria-label`.                              |
| `region`                      | Content outside a landmark                                          | New pages without `<main>` wrappers.                                            |
| `link-name` / `button-name`   | Icon-only controls without `aria-label`                             | Plane/Linear style densifies UI — icon buttons everywhere.                      |
| `nested-interactive`          | `<button>` inside `<a>` or vice versa                               | Card patterns wrap things in `<a>` and then add `<button>`.                     |
| `landmark-one-main`           | Multiple `<main>` elements or none                                  | App shell + page both render `<main>`.                                          |
| `focus-order-semantics`       | Tabindex skipping or reversed                                       | Custom dropdowns mishandle focus.                                               |

## 5. Keyboard testing

axe doesn't tab through the page. Do this manually in Playwright for critical flows:

```ts
test('can complete HITL review with keyboard only', async ({ page }) => {
  await page.goto('/extraction/123');
  await page.keyboard.press('Tab');                          // focus moves to first field
  await expect(page.locator(':focus')).toHaveAttribute('id', 'field-1');
  await page.keyboard.type('value');
  await page.keyboard.press('Tab');
  await page.keyboard.press('Enter');                         // submits
  await expect(page.getByText(/decision recorded/i)).toBeVisible();
});
```

## 6. Focus management

In Playwright:
```ts
await expect(page.locator(':focus')).toHaveAccessibleName(/submit/i);
```

In Vitest:
```ts
expect(screen.getByRole('button', { name: /submit/i })).toHaveFocus();
```

Common issues:
- Dialog opens, focus stays on the trigger (should move into the dialog).
- Dialog closes, focus is lost (should return to the trigger).
- Skip-link doesn't get focus visually when activated.

## 7. Screen-reader text

If a button is icon-only, give it an accessible name. Don't hide labels with `display: none` (also hidden from AT). Use the `sr-only` Tailwind class (already configured) or `aria-label`:

```tsx
<button aria-label="Publish run">
  <PublishIcon className="h-4 w-4" />
</button>

// or
<button>
  <PublishIcon className="h-4 w-4" />
  <span className="sr-only">Publish run</span>
</button>
```

## 8. Reduced motion

Test with prefers-reduced-motion to ensure essential UX still works without animation:

```ts
test('respects reduced motion', async ({ browser }) => {
  const context = await browser.newContext({ reducedMotion: 'reduce' });
  const page = await context.newPage();
  await page.goto('/extraction/123');
  // animations should be instant or absent; verify with a screenshot or by checking
  // for a state transition completing in <50ms.
});
```

## 9. Adding a11y to CI

It's already in some E2E flows; spread it. Convention: every new `*.ui.e2e.ts` ships with at least one axe assertion on the primary view. Adds ~500ms per flow; worth it.

## 10. When to ignore vs. fix

- **Fix immediately:** color contrast on text users actually read, missing button names, missing form labels.
- **File a ticket + disable rule:** known third-party widget violations, design-system tokens pending update.
- **Never ignore globally:** don't add rules to `axe-core.config.js` to suppress everywhere. Scope per-test.
