## Como Assinar

Para assinar este CLA, você deve:

1. Abrir um Pull Request no repositório
2. O bot CLA Assistant irá solicitar automaticamente que você assine
3. Clique no link fornecido pelo bot e siga as instruções
4. Seu PR só será mesclado após a assinatura do CLA

**Nota**: Se você trabalha para uma empresa, certifique-se de ter autorização para assinar este acordo em nome da empresa.

### Fluxo oficial de validação

Este projeto utiliza exclusivamente o [CLA Assistant](https://cla-assistant.io/) para validação de assinatura em Pull
Requests.
Não realizamos assinatura manual fora do fluxo do bot.

### Solução de problemas comuns

- **Assinei e o check ainda não atualizou**: aguarde alguns minutos e atualize a página do PR.
- **Assinei com a conta errada**: assine novamente com a mesma conta GitHub autora da contribuição.
- **PR com múltiplos autores**: cada pessoa que contribuiu com código pode precisar assinar.
- **Push direto sem PR**: abra um Pull Request, pois a verificação de CLA ocorre no fluxo de PR.

## Perguntas Frequentes

### Por que preciso assinar um CLA?

O CLA garante que o projeto possa ser licenciado de forma dual (AGPL-3.0 para uso open source e licença comercial para uso proprietário), protegendo tanto os contribuidores quanto o mantenedor.

### Isso significa que perco meus direitos autorais?

Não. Você mantém todos os direitos autorais sobre sua contribuição. O CLA apenas concede uma licença não exclusiva.

### Posso usar minha própria contribuição em outros projetos?

Sim. Como você mantém os direitos autorais, você pode usar sua contribuição em outros projetos, desde que respeite a licença AGPL-3.0 para o código do Prumo.

### E se eu não quiser assinar o CLA?

Infelizmente, não poderemos aceitar sua contribuição sem a assinatura do CLA, pois isso comprometeria a capacidade do projeto de oferecer licenças comerciais.
